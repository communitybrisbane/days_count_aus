# 🇦🇺 Days Count in AUS — 最終確定仕様書 (SPEC v2 Final)

> **正式名称**: Days Count in AUS
> **通称（アイコン下表示）**: Count
> **コンセプト**: 「Make Days Count」ー ワーホリの365日を、一生の資産（遊びと成長）に変える。
> **ターゲット**: ワーホリ渡航前・滞在中・帰国後の全フェーズのユーザー

## 3. ユーザーフェーズとカウントロジック

すべて **ユーザー端末のローカル時間（0:00〜23:59）** 基準。

| フェーズ | 表記 | 起点 | 内容 |
|---|---|---|---|
| 渡航前 (Pre-departure) | `D - 数値` | 渡航予定日 | カウントダウン |
| ワーホリ中 (In Australia) | `D + 数値` | 渡航日（当日は D+1） | 経過日数（D+365以降も継続） |
| 帰国後 (Post-return) | `R + 数値` | ステータス変更日 | 帰国後カウントアップ |

### フェーズ遷移ルール
- **手動切り替え**が基本。プロフィール画面からいつでも変更可能。
- 渡航予定日に **「ワーホリを開始しますか？」確認ダイアログ** を表示。
- D+365を超えてもカウント継続。帰国後フェーズへの切り替えはユーザー任意。

### 日付の保存形式
- `departureDate`, `returnStartDate` は **`YYYY-MM-DD`（文字列）** で保存。
- Timestamp（UTC変換）は使用しない。タイムゾーン跨ぎのバグを防止。
- カウント計算はローカル日付同士の差分で行う。

## 5. XP・レベル設計

**設計目標**: 365日毎日継続 → 帰国時 **Lv.90前後** 到達。

### XP獲得ルール

| アクション | XP | 制限 |
|---|---|---|
| ログ投稿（1日1回） | +50 | 1日1回 |
| 初投稿ウェルカムボーナス | +100 | 初回のみ（投稿50XPと合算で計150XP） |
| いいねをもらう（受信） | +10 | 無制限 |
| いいねを送る（送信） | +5 | 1日最大5回（計25XP） |
| 7日継続ボーナス | +100 | ストリークが7の倍数に達するたび（7, 14, 21...） |

### レベル計算式

```
Level = floor( sqrt( TotalXP / 2.5 ) ) + 1
```

- UI: 名前の横に **`Lv.数値`** を常時表示。
- プログレスバー: 次のレベルまでの進捗（%）を表示。

### 48時間ストリークルール
- 最終投稿から **48時間経過した瞬間** にストリーク（🔥連続投稿日数）を **0にリセット**。
- リセット **6時間前** に警告プッシュ通知。

## 6. 画面・機能詳細

### 6.1 認証・オンボーディング

**ログイン**: Googleログインのみ。

**初回登録フロー**:
1. Googleログイン
2. オンボーディング画面で以下を入力:
   - ニックネーム（**15文字以内**）
   - 渡航予定日
   - メインフォーカスモード（5つから1つ）
   - プロフィール写真（**任意** — 未設定時はイニシャル文字のカラーサークル）
3. 初回特典: 初投稿時に **ウェルカムボーナス 100XP**

**デフォルトアバター**: ニックネームの頭文字 + ユーザーIDからハッシュ生成した背景色の円。

---

### 6.2 HOME画面 (Dashboard)

- **メインカウンター**: 画面上部に巨大フォントで `D+124` 等を表示。
- **プログレスエリア**: Lv表示、次レベルまでの進捗バー、ストリーク（🔥）表示。
- **週間レポートカード**: 今週のサマリー（投稿数、獲得XP、ストリーク）をカード形式で表示。
- **運営エリア**: Firestoreの `admin_config` から取得した運営メッセージ + **JOIN ZOOMボタン**（Outback Clay色）。

---

### 6.3 EXPLORE画面（タイムライン）

- **全ユーザーの投稿**を新着順に表示。
- **無限スクロール**: Firestoreの `limit(20)` + `startAfter(lastVisibleDoc)` で20件ずつ追加読み込み。
- **フィルタ**: 5つのフォーカスモードアイコンで絞り込み。
- **いいね**: 投稿カード右下のハートボタンで送信。
- **ユーザー遷移**: 投稿者アイコンタップ → **公開版マイページ**（Lv、投稿履歴）を閲覧可能。

---

### 6.4 POST画面（ログ投稿）

- **投稿制限**: 1日1回。
- **画像**: **任意（Optional）**。
  - **画像あり**: react-easy-crop で1:1クロップ → Canvas APIで1024×1024pxにリサイズ → Firebase Storage アップロード。
  - **画像なし**: ブランドカラーのグラデーション背景にテキスト中央寄せのカードデザインで表示。
- **テキスト入力**:
  - **Fun（遊び）**: 今日一番楽しかった体験（**200文字以内**）
  - **Growth（成長）**: 今日自分のために努力した内容（**200文字以内**）
- **モード選択**: 5つのアイコンから1つ選択。

---

### 6.5 GROUPS画面（フォーカスグループ）

- **作成条件**: **Lv.7以上** のユーザーのみ。
- **最大人数**: **10名**。満員時は **「FULL」表示**。
- **識別**: 「フォーカスモードアイコン」+「グループ名」。
- **リーダー**: 作成者に **Leaderマーク** 付与。
- **参加**: 即参加（承認制なし）。
- **退出**: 自由に退出可能。
- **キック**: リーダーのみ可能。
- **リーダー退会時**: グループを **自動クローズ**（解散）。
- **機能**: テキスト送信（**100文字以内**）+ メッセージへのリアクション（いいね）。画像送信なし。

---

### 6.6 MY PAGE画面

- **プロフィール画像**: 正円表示（未設定時はイニシャルカラーサークル）。
- **アーカイブ切り替えタブ**:
  - **Timeline**: 全投稿を時系列表示
  - **Fun（遊び）**: Enjoying + Social Media の投稿を抽出
  - **Growth（成長）**: Skills + Challenging + English の投稿を抽出

---

### 6.7 設定画面 (Settings)

- **プロフィール編集**: ニックネーム、滞在地域、目標（**100文字以内**）、メインモード、渡航予定日、プロフィール写真（1:1クロップ付）。
- **フェーズ切り替え**: ステータス（渡航前/ワーホリ中/帰国後）を手動変更。
- **法定項目**: プライバシーポリシー、利用規約。
- **アカウント管理**: ログアウト、アカウント削除。
- **拡張枠**: 将来的な Stripe サブスク導入の余白。

---

### 6.8 AI振り返りデータコピー機能（週間レポート内）

毎週日曜にマイページ/HOME画面で「AI振り返り用データをコピー」ボタンを表示。

**コピー内容の構成**:
1. **【現在の目標】**: ユーザーの `goal` フィールド
2. **【直近7日間の活動ログ】**: クライアント側で集計した Fun/Growth テキスト + 各カテゴリ投稿数 + 獲得XP + ストリーク日数
3. **【AIへの指示プロンプト】**: Firestore `admin_config.ai_prompt_template` から取得

**運営メリット**: Firebase Console で `ai_prompt_template` を書き換えるだけで、アプリ更新なしにAIの分析スタイルを変更可能。

## 9. Firestore セキュリティルール方針

- **投稿の作成**: 認証済みユーザーのみ。
- **投稿の編集**: 作成者本人のみ + `editableUntil` 未経過。
- **投稿の削除**: 作成者本人のみ。
- **XP・ステータス更新**: 本人のみ書き込み可。他者は読み取り専用。
- **いいね**: 認証済みユーザーのみ作成可。自分の投稿へのいいねは不可。
- **グループメッセージ**: グループメンバーのみ読み書き可。
- **admin_config**: 読み取りは全ユーザー可。書き込みは管理者のみ。

## 10. アカウント削除時の処理

以下を一括実行:
1. Firebase Auth アカウント削除
2. `users` ドキュメント削除
3. ユーザーの全 `posts` ドキュメント削除
4. Firebase Storage 内の該当画像ファイル削除（パス: `posts/{userId}/{postId}.jpg`）
5. 参加中グループの `memberIds` から該当UID削除
6. リーダーのグループは `isClosed: true` に更新（自動クローズ）
